# Example: Weak Activation

## User Problem

Users sign up for our product but many do not complete onboarding or reach first value. The team is debating whether to redesign onboarding, add lifecycle emails, or change the product's first-run experience.

## Diagnosis

- Primary problem type: Growth Strategy / UX Improvement.
- Dominant uncertainty: Value and usability.
- Decision needed: where to intervene first to improve activation.
- Evidence level: medium if funnel data exists, low if only anecdotes exist.
- Recommended posture: diagnose, then test.

## Recommended Framework

Primary framework: Activation Framework.

Supporting frameworks:

- Journey Map to understand the first-run experience.
- Riskiest Assumption Test to validate the highest-risk intervention.

Alternatives considered:

- AARRR is useful for locating activation as the bottleneck, but it is too broad once activation is already known.
- Growth Loops should wait until users reliably reach value and retain.

## Working Canvas

### Activation Goal

Define the first value event users must reach.

### Current Path

Map signup, setup, first action, first value, and return behavior.

### Friction

Look for friction in motivation, comprehension, effort, trust, data setup, and timing.

### Segments

Separate high-intent users, low-intent users, invited users, self-serve users, and users from different acquisition channels.

### Assumptions

- Users understand what the product is for.
- The setup steps are worth the perceived effort.
- The first value event is close enough to signup.
- The product has a clear aha moment.

## Next Actions

1. Define and instrument the activation event.
2. Compare activation by segment and acquisition source.
3. Watch five users attempt onboarding.
4. Test the smallest intervention that removes the biggest friction point.
